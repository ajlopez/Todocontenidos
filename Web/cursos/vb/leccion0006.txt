Proyecto

Una aplicación Visual Basic está contenida en un proyecto. Un proyecto es un conjunto de archivos
que componen el código y la apariencia de la aplicación. Dentro de un proyecto podemos encontrar
distintos elementos: formularios, módulos de programación, y clases (veremos más adelante el concepto
de clase y objeto).

El entorno permite crear, modificar y grabar estos elementos, así como compilar el proyecto, o
ejecutarlo.

Entre los componentes de un proyecto, tenemos:

- Un archivo de proyecto, terminado en .vbp
.br
- Un archivo para cada formulario, cada ventana, terminado en .frm
.br
- Un archivo auxiliar con datos adicionales de cada formulario, terminado en .frx
.br
- Un archivo para cada módulo, terminado en .bas
.br
- Un archivo para cada clase, terminado en .cls

Hay proyectos que no tienen todos estos elementos. Lo más común, es tener
un proyecto con un formulario o varios. Los módulos tienen simple código
Basic, que no necesita tener controles visuales. Se aprovechan para poner
funciones y rutinas que podemos reusar en varios proyectos.

A partir de la versión 4, Visual Basic soporta el concepto de clases.

Hay proyectos que no tienen formularios. Es el caso de servidores .exe y
algún proyecto .dll, que pueden componerse de clases. Nuestros primeros 
proyectos serán del tipo .exe, y tendrán por lo menos un formulario.
